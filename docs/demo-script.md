# Demo Script

## Сценарий для видео-демонстрации

### Длительность: 5-7 минут

---

## Сцена 1: Введение (30 сек)

**Что показываем:**
- Главная страница платформы
- Краткое описание функционала

**Что говорим:**
> "Добро пожаловать на платформу VibeCode Jam - систему технических собеседований с ИИ-интервьюером. Сегодня мы покажем, как кандидат проходит полный цикл интервью от выбора задачи до получения финального отчёта."

---

## Сцена 2: Выбор задачи и начало интервью (1 мин)

**Что показываем:**
1. Форма выбора языка (Python/JavaScript)
2. Выбор уровня (Junior/Middle)
3. Нажатие "Начать интервью"
4. Появление задачи с описанием и примерами

**Что говорим:**
> "Кандидат выбирает язык программирования - Python, и уровень сложности - Junior. Система генерирует задачу соответствующего уровня. Вот задача: 'Написать функцию для вычисления факториала числа'. Видим описание, примеры входных и выходных данных, и шаблон кода."

**Технические детали:**
- Показать, что задача сгенерирована ИИ (можно показать в консоли запрос к API)
- Упомянуть, что задача адаптирована под уровень Junior

---

## Сцена 3: Работа в IDE (1.5 мин)

**Что показываем:**
1. Monaco Editor с шаблоном кода
2. Написание решения задачи
3. Запуск видимых тестов
4. Результаты тестов (зелёные/красные галочки)

**Что говорим:**
> "Кандидат пишет решение в браузерной IDE на базе Monaco Editor - того же редактора, что используется в VS Code. Вот решение через рекурсию. Запускаем видимые тесты - видим, что 4 из 5 тестов прошли успешно. Один тест упал - проверим, что не так."

**Технические детали:**
- Показать подсветку синтаксиса
- Показать результаты выполнения тестов
- Упомянуть, что код выполняется в изолированном Docker-контейнере

---

## Сцена 4: Исправление ошибки и общение с ИИ (1.5 мин)

**Что показываем:**
1. Исправление кода (обработка граничного случая)
2. Повторный запуск тестов - все зелёные
3. Открытие чата с ИИ
4. Вопрос от ИИ: "Можешь объяснить свой подход?"
5. Ответ кандидата
6. Ответ ИИ с анализом

**Что говорим:**
> "Видим, что упал тест для n=0. Исправляем - добавляем проверку граничного случая. Теперь все тесты проходят. Открываем чат с ИИ-интервьюером. ИИ задаёт вопрос о подходе. Кандидат объясняет использование рекурсии. ИИ анализирует ответ и даёт обратную связь, отмечая хорошее понимание базовых концепций."

**Технические детали:**
- Показать, что ИИ анализирует код в реальном времени
- Показать адаптивность - ИИ задаёт уточняющие вопросы

---

## Сцена 5: Анти-читинг (30 сек)

**Что показываем:**
1. Попытка открыть DevTools (или показать в логах событие)
2. Упоминание в интерфейсе о мониторинге

**Что говорим:**
> "Платформа отслеживает подозрительную активность: открытие DevTools, большие вставки кода, переключение вкладок. Эти события регистрируются и учитываются в финальном отчёте."

**Технические детали:**
- Можно показать в консоли браузера отправку события
- Или показать в админ-панели список событий

---

## Сцена 6: Завершение и финальный отчёт (1.5 мин)

**Что показываем:**
1. Нажатие "Завершить интервью"
2. Загрузка отчёта
3. Показ метрик:
   - Правильность: 85%
   - Оптимальность: 75%
   - Стиль кода: 90%
   - Soft-skills: 80%
4. Текстовый отзыв для кандидата
5. Переключение на вид для рекрутера

**Что говорим:**
> "Кандидат завершает интервью. Система генерирует финальный отчёт. Видим метрики: правильность решения - 85%, оптимальность алгоритма - 75%, стиль кода - 90%, soft-skills из чата - 80%. Ниже текстовый отзыв для кандидата с рекомендациями по улучшению. Переключаемся на вид для рекрутера - здесь более детальный анализ с результатами скрытых тестов и анти-чит статистикой."

**Технические детали:**
- Показать структурированный JSON отчёта (можно в отдельной вкладке)
- Упомянуть, что отчёт генерируется ИИ на основе всего контекста интервью

---

## Сцена 7: Архитектура и технические детали (30 сек)

**Что показываем:**
1. Диаграмму архитектуры (из docs/)
2. Краткий обзор компонентов

**Что говорим:**
> "Платформа построена на микросервисной архитектуре: React фронтенд, FastAPI бэкенд, отдельный Runner для выполнения кода в Docker, интеграция с Scibox LLM. Все компоненты документированы и готовы к масштабированию."

---

## Ключевые моменты для демо

1. **Рабочий end-to-end флоу** - от начала до конца без багов
2. **Адаптивность ИИ** - показать, что ИИ задаёт разные вопросы
3. **Безопасность** - упомянуть изоляцию выполнения кода
4. **Прозрачность** - показать метрики и отчёты
5. **Масштабируемость** - упомянуть архитектуру

---

## Чеклист перед демо

- [ ] Все сервисы запущены (Backend, Frontend, Runner, Database)
- [ ] Scibox API ключ настроен
- [ ] Docker работает и может создавать контейнеры
- [ ] Тестовые задачи сгенерированы
- [ ] Чат с ИИ работает
- [ ] Генерация отчёта работает
- [ ] Анти-чит события регистрируются
- [ ] Все тесты проходят
